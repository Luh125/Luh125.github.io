StringBuffer s1 = new StringBuffer();可以初始化一个新的空对象;

而代码中StringBuffer s1 = new StringBuffer(str);则是将str中字符串内容赋予s1;
这样,s1中的内容就变成了str中的内容.
s1经过revers()反转后,要将他转换成String类型,故还要用到toString(),方便进行比较.

[点1]为什么用StringBuffer而不用String呢？

SringBuffer类和String一样，也用来代表字符串.

但是相比String, StringBuffer在进行字符串处理时,不生成新的对象,在内存使用上要优于String类.
当遇到插入、删除等字符串操作时,可以考虑用StringBuffer.


[点2]关于reverse()

使用reverse()时需要用到 import java.util.*;

reverse()的作用是反转，例如字符反转和列表反转.


[点3]为什么用equals而不是”==”？

equals比较的是两个字符串内容是否相等;

==比较的是两个字符串的内存地址是否相同。

在上述代码中,我们新建了一个新对象StringBuffer s1,明显内存地址是不同的,所以要用到equals();
